# 自动修复检查清单

每次代码改动后，自动执行以下检查：

1. **清理进程和缓存**
   - 清理端口3000上的旧进程
   - 删除.next缓存目录

2. **构建检查**
   - 运行 `npm run build` 检查编译错误
   - 检查是否有颜色类引用错误
   - 检查是否有模块找不到的错误

3. **重启服务器**
   - 清理后重新启动开发服务器
   - 验证服务器是否成功启动
   - 检查端口3000是否被占用

4. **验证**
   - 检查服务器日志中的 "Ready" 或 "Local" 信息
   - 确认端口3000上有进程运行

## 常见问题修复

- **MODULE_NOT_FOUND**: 清理.next缓存并重新构建
- **端口占用**: 清理所有占用端口的进程
- **颜色类错误**: 检查tailwind.config.ts并补充缺失的颜色定义
- **EMFILE错误**: 这是文件监听器问题，不影响服务器运行
