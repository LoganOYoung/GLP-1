# 网站方向与策略：收益/运营成本最大化

目标：在**可控甚至极低运营成本**下，把站点的**收益潜力**做上去，并逐步验证、放大。

---

## 一、方向设定（Positioning）

### 1. 一句话定位

**「美国用户查 GLP-1 到手价、比选项、验药房时的首选信息与工具站。」**

- **做什么**：算成本、比替代（品牌/复方/口服）、辨合法性/短缺；不卖药、不处方。
- **不做什么**：不做诊疗建议、不做自有药房/电商、不承诺疗效；避免高合规成本的业务。

### 2. 战略重心（优先级）

| 优先级 | 重心 | 理由 |
|--------|------|------|
| 1 | **成本 + 计算器** | 搜索意图强（「GLP-1 cost」「Ozempic price」）、工具粘性高、易接联盟（telehealth/药房）。 |
| 2 | **替代 + 对比** | 承接「买不到/太贵」流量，comparison + alternatives 是联盟转化主场景。 |
| 3 | **合法性 + 短缺** | 建立信任、长尾 SEO、复访；间接提升整体转化与品牌。 |

**方向结论**：以「成本与替代」为**变现主入口**，以「合法性/短缺」为**信任与长尾流量**，保持纯信息+工具、低合规成本。

---

## 二、收益策略（Revenue）

### 1. 主收益：联盟（Affiliate）

- **现状**：telehealth、药房/资格链接已接，Footer + 近链披露已做。
- **策略**：
  - **集中优化高意图页**：Calculator 结果、Alternatives（Telehealth + 药房列表）、Legitimacy 验证药房。确保这些页的 CTA 清晰、披露到位。
  - **按转化数据迭代**：看哪些合作方、哪些页面带来点击/转化，加大曝光与内链，弱化无效位。
  - **分散合作方**：再接 1～2 家 telehealth 或药房联盟，分摊政策风险、测试不同佣金。
- **不加成本**：不增人力，只调页面与链接结构。

### 2. 次收益：赞助/品牌内容（可选、后期）

- 在**不破坏信任**的前提下，可接「赞助问答」「赞助对比位」（明确标注 Sponsored）。
- 仅考虑与现有受众一致的品牌（GLP-1 相关、合规），单价高、量不必大，保持边际成本低。

### 3. 暂不做的（控制成本与合规）

- **自有付费工具/会员**：需持续维护、客服、定价，成本高；当前阶段用免费工具+联盟更划算。
- **Lead 贩卖**：合规与品牌风险大；若以后做 lead gen，应以「用户授权、单一合作方、明确用途」为主，而不是卖名单。
- **展示广告（AdSense 等）**：单价低、体验差、与医疗内容易冲突；除非流量极大否则不优先。

**收益结论**：**联盟为主、赞助为辅；优先把现有高意图页的转化做高，再考虑新合作方与赞助。**

## 三、成本策略（Operating Cost）

### 1. 内容与数据：极简化、自动化

| 项目 | 策略 | 成本 |
|------|------|------|
| 价格/政策数据 | 用现有 `data/*.json` + 每月/每季人工或 Action 更新；可选外部 API 拉取（已留脚本）。 | 低：少量人工或零（若全自动）。 |
| 新内容 | 以「问题—直接答案」的短篇为主，每 1～2 周 1 篇，聚焦成本/替代/合法性长尾词。 | 低：可自己写或外包单篇。 |
| 避免 | 不做日更、不做新闻站、不做 UGC/评论，避免审核与合规成本。 | 省人力与法律风险。 |

### 2. 技术与运维：零后端、静态优先

- **保持**：纯静态导出（Next `output: 'export'`），Vercel 托管，无自建服务器、无数据库。
- **自动化**：GitHub Actions 每月刷新 lastUpdated、可选拉取外部数据；部署随 push 自动完成。
- **不扩张**：不接重后端（登录、支付、复杂 API），除非单点功能能明确带来收益且成本可控。

### 3. 合规与法律：预防为主

- **已有**：免责、隐私/条款概要、联盟披露、医疗信息边界清晰。
- **策略**：重大变更（新变现、新数据收集）前做一次合规检查；内容上不承诺疗效、不替代医生，把合规成本压在「一次性+小幅迭代」上。

### 4. 获客：SEO 为主、付费试水

- **主渠道**：SEO（站内结构已有基础），把「成本/替代/合法性」三个入口 + 计算器 + 长尾专题做扎实；外链随内容与引用自然增长，不买大量垃圾外链。
- **付费**：仅在验证过「某类关键词转化好」后，用小预算测试 Google/Bing 搜索广告，ROI 为正再放大。
- **避免**：不铺大量社媒账号、不搞社群运营，除非单渠道已证明能带来可衡量转化。

**成本结论**：**内容与数据极简+自动化，技术零后端，合规预防式，获客以 SEO 为主、付费为辅。** 目标是把「运营成本」压到几乎只有：域名+托管+少量内容/数据更新+偶尔合规检查。

## 四、策略矩阵：做什么 / 不做什么

| 维度 | 多做 / 优先做 | 少做 / 暂不做 |
|------|----------------|----------------|
| **内容** | 成本/替代/合法性总纲页优化；FAQ 直接答；少量长尾专题。 | 日更、新闻、UGC、大篇幅教程。 |
| **工具** | 维护并优化 Calculator、Comparison、Quiz、Dose Converter；保证可用与转化路径。 | 新增大而全的付费工具、会员体系。 |
| **变现** | 联盟（telehealth、药房）；高意图页 CTA 与披露。 | 卖 lead、泛展示广告、未标注赞助。 |
| **技术** | 静态站、Vercel、Actions 自动化、结构化数据。 | 自建后端、数据库、复杂 API。 |
| **获客** | SEO、内链与入口清晰、Featured Snippet 优化。 | 买外链、铺社媒、大预算品牌广告。 |
| **合规** | 免责、披露、隐私/条款、不越界医疗建议。 | 承诺疗效、卖药、收集敏感健康数据。 |

## 五、阶段目标与检验（OKR 风格）

### 阶段 1：验证（上线后 3～6 个月）

- **收益**：有稳定联盟点击/转化（哪怕金额小），能分出哪些页、哪些合作方有效。
- **成本**：月度运营成本接近「域名+托管+零或极低人力」。
- **检验**：Search Console 有「GLP-1 cost / alternatives / legitimacy」相关词排名与点击；联盟后台有转化记录。

### 阶段 2：放大（6～12 个月）

- **收益**：联盟收入可覆盖成本并有余；可选 1 个赞助或新联盟渠道试水。
- **成本**：仍不招人、不建后端；内容/数据更新保持轻量或自动化。
- **检验**：主要入口页（cost-insurance、alternatives、legitimacy、calculator）自然流量与转化同比明显上升。

### 阶段 3：稳定与可选扩展（12 个月+）

- **收益**：联盟稳定；若单页或单渠道验证成功，可试 lead gen 或赞助专区。
- **成本**：仅在「新增收益明显大于新增成本」时加人、加功能。
- **检验**：单位流量收益（RPM 或 per-session value）提升，或总利润增长而成本增速更低。

---

## 六、一句话总结

**方向**：做「美国 GLP-1 成本 / 替代 / 合法性」的**首选信息与工具站**，不越界医疗、不卖药。  
**收益**：以**联盟为主**，优化高意图页转化，再考虑赞助与少量新渠道。  
**成本**：**内容与数据极简+自动化，技术零后端，获客以 SEO 为主**，把运营成本压到最低。  
**策略**：只做能直接带来流量或转化、且维护成本低的事；不做重后端、重运营、高合规风险的事。这样收益/运营成本比才能持续最大化。
