# 首页规划：转化工具 × 医疗信任

RxLikewise.com 首页作为 GLP-1 决策漏斗入口，需同时提升**转化工具**与**医疗信任**。本文档基于现状评价与规划建议整理，供产品与研发对齐。

---

## 一、现状评价

### 优点（已具备）

| 维度 | 现状 |
|------|------|
| 直击痛点 | Hero 明确点出「价格高、保险拒绝、药物短缺」，精准对应美国 GLP-1 用户焦虑。 |
| 工具导向 | 首屏即「成本计算器」「选择路径测验」，非博客式堆内容，参与度高。 |
| 用户分流 | 「有保险 / 无保险 / 迷茫」三入口，快速分流高意向用户。 |
| 2026 实时性 | TrumpRx、口服新药等政策与产品更新，传递「网站是活的」。 |

### 不足（待补）

| 维度 | 缺口 |
|------|------|
| 视觉权威感 | UI 偏工具站，缺乏 WebMD/Mayo 式权威、厚重医疗感。 |
| 社交证明 | 无用户成功案例（如「用模板成功报销」）、无真实药房评价。 |
| 互动与粘性 | 缺「缺货地图」「价格走势」等动态展示，用户回访动机弱。 |

**结论**：首页功能完整、痛点清晰，约 **75 分**；要上 **95 分**，需从「信息提供者」升级为「利益捍卫者」。

---

## 二、首页五板块规划（目标：千万美金级入口）

### 1. 强化 Hero：实时数据看板（第一屏）

- **规划**：在现有 CTA 旁增加**「实时数据看板」**。
- **元素示例**：  
  - 「今日全国 GLP-1 平均价格：$XXX」  
  - 「本周保险申诉成功率：78%」
- **目的**：一进站即传递「数据驱动、专业平台」。
- **实现依赖**：需价格/申诉数据源（API 或人工周更）；无数据时可先做**静态占位 + “Data as of [date]”**，后续接真实数据。

---

### 2. 保险导航模块（中层）

- **规划**：不只给链接，增加**「搜索你的保险公司」**选择框。
- **交互**：用户选「BlueCross」→ 弹出「该保险最近对 Zepbound 审批趋严，建议下载 X 模板」等个性化提示。
- **价值**：强个性化、提高模板下载与申诉页转化。
- **实现**：前端可先做「保险公司下拉 + 静态文案/模板链接」；精细化需后端或 CMS 维护「保险公司 × 药品 × 建议」规则。

---

### 3. 安全与合法性验证专区（中层）

- **规划**：**「一键验真」**：用户输入某低价网站 URL，系统给出「红旗提醒」或「认证标识」。
- **价值**：树立「合规裁判」角色，增强信任与分享。
- **实现**：需 URL 规则库/名单（已知诈骗站 vs 已知合规站）+ 简单后端或 Edge Function；首版可做「输入 URL + 展示通用红旗清单 + 建议查州药房委员会」不验真，后续再接真实校验。

---

### 4. 平替方案对比矩阵（下层）

- **规划**：横向对比**品牌药 vs 复合药 vs 最新口服药**（价格、有效性、获取难度），每方案下 **「Check Availability」** 导流合作伙伴。
- **现状**：首页已有三路径卡片（Brand/Compounded/Oral），Alternatives 有 PathComparisonTable。
- **增强**：首页可增加**紧凑版对比表** + 明确「Check Availability」按钮，与现有 `/alternatives` 和合作方链接打通。

---

### 5. 社区与实时动态（底层）

- **规划**：**「近期获药成功」**滚动条，例：「3 分钟前，休斯顿用户通过 X 药房拿到 Wegovy」。
- **目的**：制造实时感与希望，断货期促日活。
- **实现**：需用户上报或药房/合作方数据；无数据时可先做**静态「近期动态」示例 + 文案说明**，或接入 UGC/轻量后端后再上真实流。

---

## 三、总结建议（产品侧）

- **UI**：增加深蓝、医疗绿，考虑医生/专家背书视觉（照片、引用）。
- **交互**：优先「实时价格/申诉成功率」展示（哪怕先静态后接 API）。
- **内容**：做「2026 GLP-1 政策百科」，把 TrumpRx 等讲透，争取该政策下的**第一解释权**。
- **护城河**：在 2026 年，**实时、可预测的价格与货源信息**比静态内容更有价值，首页应逐步向「数据驱动的实时性」靠拢。

---

## 四、实现优先级与可行性（研发视角）

| 板块 | 优先级 | 前端可做（无/轻后端） | 需后端/数据 |
|------|--------|------------------------|-------------|
| Hero 实时看板 | P0 | 占位区块 + 静态数字 + “Data as of” | 真实价格/申诉成功率数据源 |
| 保险导航 | P0 | 保险公司下拉 + 静态提示 + 模板链接 | 精细化规则可 CMS/后端 |
| 安全验真 | P1 | 输入 URL + 通用红旗说明 + 外链州药房委员会 | URL 黑白名单与校验逻辑 |
| 平替对比矩阵 | P1 | 首页紧凑对比表 + Check Availability 按钮 | 合作方链接已有可复用 |
| 社区实时动态 | P2 | 静态示例或假数据滚动 | 用户/药房上报或 API |

**建议首期**：  
- 实现 **Hero 实时看板占位**（文案 + 静态数字 + 数据日期），为后续接 API 留位。  
- 实现 **保险导航**：首页或 Cost 入口增加「选保险公司 → 展示对应提示与模板链接」。

如需，可再拆成具体 ticket（文案、组件名、路由、数据格式）便于排期。
